## High

### 1. BUG-T1-001 Неверный значок очистки поля ввода

**Название:** Вместо крестика очистки поля отображается значок «битой» картинки

**Окружение:**

- Платформа: Web (desktop)
- Страница: поиск вакансий по запросу «Аналитик»

**Шаги воспроизведения:**

1. Перейти на страницу поиска вакансий «Аналитик».
2. Ввести любой текст в поле поиска.
3. Посмотреть на иконку справа от поля ввода.

**Фактический результат:**

- Вместо ожидаемого обозначения очистки (крестик) показывается иконка неопознанного изображения (зачёркнутый файл).

**Ожидаемый результат:**

- Отображается корректная иконка очистки поля (крестик) в соответствии с дизайном‑системой.

**Приоритет:** high

**Обоснование приоритета:**

- Создаётся впечатление сломанного интерфейса (битая картинка).
- Непонятно, что по клику поле очистится.

### 2. BUG-T1-002 Фильтр «Направление: DataScience» не ограничивает результаты

**Название:** Игнорирование фильтра направления «DataScience» (показываются другие направления)

**Окружение:**

- Платформа: Web (desktop)
- Страница: поиск вакансий по запросу «Аналитик»

**Шаги воспроизведения:**

1. В фильтре «Направление» выбрать «DataScience».
2. Применить фильтр.
3. Просмотреть список направлений и вакансий на странице.

**Фактический результат:**

- Помимо блока DataScience отображаются направления «Клиентский сервис», «Финансы, закупки и аудит», «Аналитика данных»,
  «Стратегия и M&A».

**Ожидаемый результат:**

- При выборе «DataScience» должны отображаться только вакансии и блоки, относящиеся к направлению DataScience.

**Приоритет:** high

**Обоснование приоритета:**

- Пользователь получает «шумную» выдачу и тратит время на ручное отфильтровываниелишних направлений
- Возникает ощущение, что фильтры не работают

### 3. BUG-T1-003 Чек-бокс «Руководитель» не применяется к результатам

**Название:** Чек-бокс по признаку «Руководитель» не отрабатывает

**Окружение:**

- Платформа: Web (desktop)
- Страница: поиск вакансий по запросу «Аналитик»

**Шаги воспроизведения:**

1. Установить чекбокс «Руководитель».
2. Обновить выдачу.
3. Проверить названия и описания отображаемых вакансий.

**Фактический результат:**

- В выдаче присутствуют как руководящие, так и обычные позиции.

**Ожидаемый результат:**

- При активном чек-боксе должны отображаться только позиции с руководящим уровнем.

**Приоритет:** high

**Обоснование приоритета:**

- Кандидат, который ищет только руководящие роли, вынужден просматривать нерелевантные вакансии.
- Снижается доверие к качеству фильтрации.

### 4. BUG-T1-004 Плохая читаемость фильтров (контраст/кегль)

**Название:** Низкая читаемость текста в панели фильтров поиска вакансий

**Окружение:**

- Платформа: Web (mobile)
- Страница: поиск вакансий по запросу «Аналитик»

**Шаги воспроизведения:**

1. Открыть страницу поиска по запросу «Аналитик».
2. Обратить внимание на область фильтров под заголовком (город, направления, дополнительные фильтры).

**Фактический результат:**

- Подписи и значения в селектах (серый текст) визуально очень бледные на белом фоне.
- Кегль текста в некоторых контролах заметно меньше основного текста.

**Ожидаемый результат:**

- Текст фильтров имеет достаточный контраст с фоном.
- Кегль не меньше минимально рекомендуемого для интерфейсов и согласован с остальной типографикой.

**Приоритет:** high

**Обоснование приоритета:**

- Сложно понять, какие фильтры уже применены.
- Повышается вероятность ошибок при поиске.

### 5. BUG-T1-005 Сообщение «Ничего не нашлось» при наличии результатов

**Название:** Отображение «Ничего не нашлось» при наличии выдачи по запросу

**Окружение:**

- Платформа: Web (desktop)
- Страница: поиск вакансий по запросу «Аналитик»

**Шаги воспроизведения:**

1. Выполнить поиск по запросу «Аналитик» с текущими фильтрами.
2. Убедиться, что на странице есть блоки с вакансиями.
3. Найти надпись «Ничего не нашлось» между блоками.

**Фактический результат:**

- Одновременно с реальными результатами показывается крупная надпись «Ничего не нашлось», не привязанная явно к
  конкретному фильтру или группе.

**Ожидаемый результат:**

- Сообщение «Ничего не нашлось» отображается только когда действительно нет ни одной подходящей вакансии, либо строго
  внутри пустой категории с понятным контекстом.

**Приоритет:** high

**Обоснование приоритета:**

- Пользователь может решить, что поиск не дал результатов, и покинуть страницу, даже не просматривая имеющиеся вакансии.

### 7. BUG-T1-007 Неверно указанный адрес локации

**Название:** Ошибка в адресе офиса в Москве: вместо «ул. Лесная, 7» отображается «ул. Лесная, 77»

**Окружение:**

- Платформа: Web (mobile/desktop)
- Страница: поиск вакансий по запросу «Аналитик»

**Шаги воспроизведения:**

1. Открыть страницу поиска вакансий «Аналитик».
2. Найти вакансию(и), где в описании/карточке в качестве адреса указана «ул. Лесная,77».
3. Сравнить отображаемый адрес с фактическим адресом офиса/локации.

**Фактический результат:**

- В карточке вакансии указан адрес «ул. Лесная,77».

**Ожидаемый результат:**

- В карточке вакансии указан корректный адрес «ул. Лесная, 7».

**Влияние на пользователя:**

- Кандидаты могут приехать по неправильному адресу.
- Повышается вероятность опозданий на интервью, недовольства и репутационных рисков для компании.

**Приоритет:** medium

### 8. BUG-T1-008 Мелкий шрифт в списке направлений и вакансий

**Название:** Слишком мелкий кегль в блоках направлений и карточках вакансий

**Окружение:**

- Платформа: Web (mobile)
- Страница: поиск вакансий по запросу «Аналитик»

1. Выполнить поиск «Аналитик».
2. Пролистать до блоков направлений и карточек внутри них.

**Фактический результат:**

**Приоритет:** medium

**Обоснование приоритета:**

### 9. BUG-T1-009 Несовпадение размеров кнопок подразделов

**Название:** Нерегулярный размер кнопки у подраздела «Поддержка пользователей»

2. Сравнить внешний вид подраздела «Поддержка пользователей» с соседними.

**Приоритет:** low

**Обоснование приоритета:**

- Возникает визуальный шум и ощущение неконсистентного интерфейса.
